## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue. Instead, please report it responsibly:

1. **Email**: [Your security email]
2. **Subject**: "Security Vulnerability in TCP-AMP"
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours and work with you to address the issue.

## Security Best Practices

### For Users

- ✅ Always run with appropriate permissions
- ✅ Use only on systems you own or have authorization to test
- ✅ Keep the tool updated
- ✅ Review code before compiling
- ✅ Use in isolated test environments

### For Developers

- ✅ Review all code changes
- ✅ Test security implications
- ✅ Follow secure coding practices
- ✅ Keep dependencies updated

## Responsible Disclosure

We follow responsible disclosure practices:

1. Report the vulnerability privately
2. Allow time for a fix to be developed
3. Coordinate public disclosure
4. Credit the reporter (if desired)
