# Contributing to TCP-AMP

Thank you for your interest in contributing to TCP-AMP!

## Code of Conduct

- Be respectful and professional
- Focus on constructive feedback
- Follow security best practices
- Remember this tool is for authorized testing only

## How to Contribute

### Reporting Issues

1. Check if the issue already exists
2. Provide detailed information:
   - OS and version
   - Compiler version
   - Steps to reproduce
   - Expected vs actual behavior

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

### Code Style

- Follow existing code style
- Use meaningful variable names
- Add comments for complex logic
- Keep functions focused and small
- Test your changes before submitting

### Testing

- Test on your own infrastructure only
- Verify compilation on Linux
- Test with different thread counts
- Verify statistics output
- Test BGP amplification mode
- Test SOCKS5 proxy mode

## Security Considerations

- Never commit credentials or sensitive data
- Review code for security vulnerabilities
- Follow responsible disclosure for security issues

## Questions?

Open an issue for questions or discussions.
